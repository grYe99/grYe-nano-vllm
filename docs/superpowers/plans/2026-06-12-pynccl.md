# PyNccl Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace `torch.distributed.all_reduce` with a direct ctypes NCCL wrapper (PyNccl) to eliminate ~80us CPU dispatch overhead per all_reduce call.

**Architecture:** Module-level singleton `NcclCommunicator` wrapping NCCL via ctypes, initialized in `ModelRunner` via `dist.broadcast_object_list` bootstrapping, and accessed by `RowParallelLinear`/`VocabParallelEmbedding`. Supports both eager mode and CUDA Graph capture natively.

**Tech Stack:** Python ctypes, libnccl.so (NCCL 2.27.7), CUDA 12.9, PyTorch 2.8

```python
# nanovllm/utils/pynccl.py
import ctypes
import torch

_NCCL_COMM = None

# ncclResult_t constants
_NCCL_OK = 0
_nccl_sum = 0
_nccl_float = 0
_nccl_float16 = 1
_nccl_bfloat16 = 2
_nccl_int32 = 3

# ncclUniqueId struct (128-byte opaque buffer)
class NcclUniqueId(ctypes.Structure):
    _fields_ = [("internal", ctypes.c_uint8 * 128)]


def _check(result: int) -> None:
    if result != _NCCL_OK:
        raise RuntimeError(f"NCCL error code: {result}")


def init_communicator(world_size: int, rank: int, unique_id: bytes) -> None:
    global _NCCL_COMM
    _NCCL_COMM = NcclCommunicator(world_size, rank, unique_id)


def get_communicator() -> "NcclCommunicator":
    assert _NCCL_COMM is not None, "NcclCommunicator not initialized"
    return _NCCL_COMM


class NcclCommunicator:

    def __init__(self, world_size: int, rank: int, unique_id: bytes):
        self.world_size = world_size
        self.rank = rank
        self._lib = ctypes.CDLL("libnccl.so")
        self._init_nccl_types()
        self._setup_argtypes()
        # Parse unique_id bytes into ncclUniqueId struct
        uid = NcclUniqueId()
        ctypes.memmove(ctypes.byref(uid), unique_id, 128)
        comm = ctypes.c_void_p()
        _check(self._lib.ncclCommInitRank(
            ctypes.byref(comm), ctypes.c_int(world_size),
            uid, ctypes.c_int(rank)))
        self._comm = comm

    def _init_nccl_types(self) -> None:
        self._dtype_map: dict[torch.dtype, int] = {
            torch.float32: _nccl_float,
            torch.float16: _nccl_float16,
            torch.bfloat16: _nccl_bfloat16,
            torch.int32: _nccl_int32,
        }

    def _setup_argtypes(self) -> None:
        # ncclAllReduce: sendbuff, recvbuff, count, datatype, op, comm, stream
        self._lib.ncclAllReduce.argtypes = [
            ctypes.c_void_p, ctypes.c_void_p, ctypes.c_size_t,
            ctypes.c_int, ctypes.c_int, ctypes.c_void_p, ctypes.c_void_p,
        ]
        self._lib.ncclAllReduce.restype = ctypes.c_int
        # ncclCommDestroy: comm
        self._lib.ncclCommDestroy.argtypes = [ctypes.c_void_p]
        self._lib.ncclCommDestroy.restype = ctypes.c_int

    def all_reduce(self, tensor: torch.Tensor) -> None:
        stream = torch.cuda.current_stream()
        stream_ptr = ctypes.c_void_p(stream.cuda_stream)
        _check(self._lib.ncclAllReduce(
            tensor.data_ptr(),
            tensor.data_ptr(),
            ctypes.c_size_t(tensor.numel()),
            ctypes.c_int(self._dtype_map[tensor.dtype]),
            ctypes.c_int(_nccl_sum),
            self._comm,
            stream_ptr,
        ))

    def __del__(self) -> None:
        if hasattr(self, "_comm") and self._comm:
            self._lib.ncclCommDestroy(self._comm)
```

Add `import ctypes` to the module-level imports.

Note: `dist.broadcast_object_list` is used instead of the existing SharedMemory IPC because:
- The shared memory is set up AFTER model creation (warmup uses all_reduce, so PyNccl must be ready before warmup)
- `dist.broadcast_object_list` works over the already-initialized NCCL process group
- Avoids the chicken-and-egg problem of needing shared memory before model ops

### Task 5: Write learning document

**Files:**
- Create: `docs/pynccl_learning.md`

- [ ] **Step 1: Create learning document**

`docs/pynccl_learning.md` covers these sections:

1. **Background: TP and the all_reduce bottleneck**
   - Why TP needs all_reduce at each RowParallelLinear
   - Decode step: 48 all_reduces per step (2/layer × 24 layers)
   - Profiler trace from 2x RTX 4090 — raw data table with cycle counts

2. **torch.distributed all_reduce call chain**
   - `dist.all_reduce(tensor)` Python API
   - ProcessGroupNCCL (C++): `record_param_comms`, communicator lookup
   - cuLaunchKernel → NCCL kernel on GPU
   - Per-call cost breakdown table: 97us CPU vs 8us GPU (12x ratio)
   - Source of overhead: GIL handling, profiler events, C++ dispatcher setup, synchronization

3. **PyNccl design**
   - ctypes binding to libnccl.so
   - ncclAllReduce directly with stream handle from `torch.cuda.current_stream()`
   - What's eliminated: c10d wrapper, profiler hooks, dispatcher
   - Remaining cost: cuLaunchKernel + actual GPU kernel (~17us total target)

4. **CUDA Graph compatibility**
   - During `torch.cuda.graph` capture: NCCL kernel launched on capture stream
   - During replay: captured NCCL kernel replayed from graph, zero CPU overhead
   - NCCL 2.19+ supports CUDA graph capture (user has 2.27.7)
   - No special handling required — stream management is automatic

5. **PCIe vs NVLink constraint**
   - NVLink: GPU-initiated P2P transfers, custom all-reduce kernels possible (vLLM's approach)
   - PCIe: CPU-mediated DMA, every transfer requires CPU kernel launch
   - Why vLLM's CustomAllReduce doesn't work on PCIe — fundamental bus architecture limitation
   - PyNccl is the correct optimization for PCIe: reduce CPU dispatch, not GPU kernel time

6. **Comparison: torch.distributed vs PyNccl**

| Metric | torch.distributed | PyNccl | Improvement |
|--------|-------------------|--------|-------------|
| CPU time per call | ~97us | ~20us (est.) | ~5x |
| GPU time per call | ~8us | ~8us | Same |
| Calls per decode step | 48 | 48 | Same |
| CPU time per step | ~4.7ms | ~1ms (est.) | ~4x |
| CUDA Graph support | Yes | Yes | Same |
